# 工作日志规则（每次会话都执行，不要等我提醒）

在项目根目录维护 `review.md`。每当完成以下任一动作后，立即在 review.md 末尾追加一条记录：
- 修复了一个 bug
- 做出影响代码结构的设计决策
- 放弃某个方案换成另一个
- **修改数据模型**（Entity、数据库表、核心数据类）
- **涉及隐私/权限的决策**（数据存储位置、加密方式、权限申请、网络请求）

## 记录格式
```
### YYYY-MM-DD HH:MM  [标签]
**问题/背景**：1-2 句说明遇到什么
**方案**：为什么这么做，有没有备选
**改动**：
- `path/to/file.kt`: 简述改了什么
```

标签用 `[BUG]` `[设计]` `[数据模型]` `[隐私]` `[重构]` 等，方便以后 Ctrl+F 筛选。

## 规则
- 时间按正序追加（新的在下面）
- 一次改动涉及多个独立问题时，拆成多条
- 不要等会话结束才补记
- 数据模型变更必须额外注明：是否需要数据库迁移
